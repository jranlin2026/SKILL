# Brainstorming

Use this skill when the user is still shaping an idea and needs productive exploration before implementation.

## Workflow

1. Restate the goal in one sentence.
2. Identify the decision type: product direction, workflow design, feature scope, data model, UI direction, go-to-market, operations, or technical approach.
3. Generate 3-5 distinct options, not minor variations.
4. For each option, give:
   - Best fit
   - Tradeoff
   - Risk
   - What to test first
5. Recommend one direction when enough context exists.
6. Convert the chosen direction into next actions, a PRD outline, or implementation tasks if the user asks to proceed.

## Style

- Prefer concrete business examples over abstract frameworks.
- For CRM/CMR/internal systems, include roles, permissions, workflow states, data ownership, exceptions, and audit needs.
- Avoid generic "could be anything" lists. Give a point of view.
- When the user is deciding between options, use a compact comparison table.

## Useful Prompts

- "帮我脑暴这个 CRM 模块应该怎么设计。"
- "这个功能有几种做法？你推荐哪种？"
- "把这个想法拆成可落地的产品方案。"
- "站在老板、销售、财务、交付几个角色分别想一下。"
