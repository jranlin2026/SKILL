# Taste Skill

Use this skill to raise the visual quality of a product, especially when the user asks for aesthetic judgment.

## Evaluation Lens

Judge the design by:

- Information hierarchy
- Visual density
- Typography
- Alignment and spacing
- Color discipline
- Interaction clarity
- Domain fit
- Whether it feels like a real product or a template

## For CRM/CMR/Admin Systems

Good internal systems should feel:

- Calm
- Dense but readable
- Operational
- Trustworthy
- Fast to scan
- Built for repeated daily use

Avoid:

- Overly large hero sections
- Marketing-page layouts
- Decorative gradient blobs
- Neon "AI dashboard" styling
- Too many cards inside cards
- Low-density pages that waste operational space

## Critique Format

When reviewing a UI, give:

1. Overall direction
2. What works
3. What feels weak
4. Priority fixes
5. Concrete redesign suggestions

Be opinionated. Do not hide behind "it depends" when there is enough context.

## Polish Checklist

- Is the primary action obvious?
- Can the user scan the page in 5 seconds?
- Are status colors meaningful?
- Are tables and forms usable at real data density?
- Does text fit without wrapping badly?
- Does the page suit the business domain?
